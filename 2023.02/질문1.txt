1/27
java 쓰레드
Q 멀티코어 멀티쓰레드 사용 시 여러 쓰레드를 동시에 작업하는 경우가 발생하는데 
  이게 어떻게 가능한건지? 이러한 현상도 교착상태(DeadLock)라고 하는지?
A 코어를 작업자라고 하면, 멀티 코어는 작업자가 여러명임을 뜻함, 시분할 방식으로
  여러 작업을 두 작업자가 작업하므로, 둘 다 한가지 작업을 하게되는 겸칩 부분이
  발생하게 되는것이다, 이러한 경우, 서로 간에 다른 작업자가 일을 마친 뒤에 작업하려
  고 기다리게 되는데 이러한 현상을 deadlock 교착상태라고 한다.

Java 컴파일 및 실행과정
Q 심볼 레퍼런스란?
A 심볼릭 레퍼런스는 컴파일 시 참조변수에 대입되는 임시 주소정도로 생각하면된다.
   Runtime 시 JVM에 의해서 실제 주소값인 다이렉트 레퍼런스로 변경된다.

열거형
Q 열거형 사용 시 내부 상수의 값을 변경하여도 컴파일이 다시 실행되지는 않게 되는데
   이는 왜 이런건가요?  A 이넘 클래스 내부 상수는 객체이므로, 동적으로 변경이 가능함.
Q 그럼 정적과 동적의 차이는 시점인건가요?
A 컴파일 시점이 정적인 시점, Runtime 시점이 동적인 시점.
